## Phase 1: Foundation (Skeleton + Config + Routing + Proxy) — COMPLETE

- Task: Project init, config loading, routing, reverse proxy, middleware chain, graceful shutdown
- Key decisions:
  - Single external dep: gopkg.in/yaml.v3; all else stdlib
  - Env var substitution on raw YAML text before parsing
  - regexp.MustCompile at startup (fail fast)
  - One http.Transport per route for connection pooling
  - RequestID middleware sets header on both request (for upstream propagation) and response
  - No 405 — router returns 404 when method doesn't match, falls through to next route
  - Recovery middleware catches panics, returns 500 JSON
  - UUID v4 via crypto/rand, no external dep

## Phase 2: Observability + Logging — COMPLETE

- Task: Structured JSON logging, /health endpoint, Prometheus metrics
- Key decisions:
  - New external dep: github.com/prometheus/client_golang@v1.20.0
  - Shared WrappedResponseWriter in middleware pkg (idempotent wrapping, captures status+bytes)
  - Logging middleware uses slog; Debug level logs headers, Info logs core fields
  - Route name extracted from context via defer (router sets it during handler execution)
  - Health endpoint bypasses middleware chain, registered directly on mux
  - Health returns "degraded" if Redis checker fails, "up" otherwise; always 200
  - Metrics: 7 Prometheus metrics registered (counter, histogram, gauge); CB/ratelimit/upstream metrics registered now but populated in future phases
  - Each test uses own prometheus.NewRegistry() for isolation
  - Used collectors.NewProcessCollector/NewGoCollector (non-deprecated API)
- Files created:
