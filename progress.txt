## Phase 4: Authentication — COMPLETE

- Task: Auth middleware with JWT, API Key, and OIDC strategies behind common Authenticator interface
- Key decisions:
  - New external dep: github.com/golang-jwt/jwt/v5@v5.3.1
  - Authenticator interface with factory dispatch on provider.Type (jwt/apikey/oidc)
  - Auth middleware reads MatchedRoute from context → looks up provider name → calls Authenticate
  - Provider "none" or nil auth config → skip auth entirely
  - JWT: HMAC/RSA/ECDSA support; key material loaded once at construction; Bearer token from Authorization header
  - API Key: constant-time compare via crypto/subtle.ConstantTimeCompare; configurable header (default X-API-Key)
  - OIDC: JWKS fetching with background refresh (80% of TTL); JWK→RSA/ECDSA parsing; introspection mode via RFC 7662
  - OIDC discovery: fetches .well-known/openid-configuration for jwks_uri if only issuer_url set
  - AuthResult stored in context for downstream middleware (rate limit claim keying)
  - Error responses: JSON {"error":"unauthorized","message":"..."} format
  - JWT error classification: expired/signature/generic mapped to specific messages
- Files created:
  - internal/middleware/auth/auth.go (interface, factory, middleware, context helpers, writeError)
  - internal/middleware/auth/auth_test.go (8 tests)
  - internal/middleware/auth/jwt.go (JWTAuthenticator, key loading, classifyJWTError)

## Phase 3: CORS — COMPLETE

- Task: CORS preflight handling, response header injection, per-route override
- Key decisions:
  - No external deps; stdlib only
  - corsResponseWriter intercepts WriteHeader to inject CORS headers at write time (after proxy copies upstream headers)
  - Per-route override uses isOverride flag + internal marker header (X-Gateway-Cors-Override) to prevent global CORS from overwriting; marker deleted before response sent
  - Complete replacement semantics: per-route CORS fully replaces global, no field-level merge
  - Wildcard + credentials edge case: logs warning at construction, uses requesting origin instead of *
  - Preflight = OPTIONS + Origin + Access-Control-Request-Method (all three required); plain OPTIONS forwarded to handler
  - CORS positioned last in global chain (after Metrics, before Router)
  - Preflights use global config only (router hasn't matched yet); per-route overrides apply to regular requests via per-route handler chain
  - Added ExposedHeaders field to CORSConfig, CORS pointer to RouteConfig
- Files created:
  - internal/middleware/cors.go (CORS, CORSOverride, corsResponseWriter, helpers)

## Phase 5: Rate Limiting — COMPLETE

- Task: Per-route rate limiting with in-memory token bucket and Redis sliding window backends
- Key decisions:
  - New external dep: github.com/redis/go-redis/v9@v9.18.0
  - Limiter interface: Allow(ctx, key, limit, window) → (allowed, remaining, resetAt, err)
  - Key extraction: ip (XFF→RemoteAddr), header:<name>, claim:<name> with IP fallback + slog.Warn
  - Memory backend: token bucket with sync.Map + per-bucket mutex; background cleanup goroutine (60s interval, 5min eviction TTL); context-aware shutdown
  - Redis backend: Lua sliding window script (ZSET); atomic check+add; key auto-expiry via EXPIRE
  - Fail open on backend errors: Redis outage → log error, allow request
  - Rate limit positioned before auth in global chain (prevents unauthenticated flood)
  - Composite key: routeName + ":" + extractedKey
  - 429 response: JSON body {error, message, retry_after} + Retry-After header
  - X-RateLimit-{Limit,Remaining,Reset} headers set on every response for rate-limited routes
  - gateway_rate_limit_rejected_total counter incremented on 429
  - Redis health check wired into /health endpoint (shows degraded if Redis down)
  - Redis client closed on graceful shutdown
- Files created:
  - internal/middleware/ratelimit/ratelimit.go (Limiter interface, RateLimit middleware, extractKey, extractIP)
  - internal/middleware/ratelimit/ratelimit_test.go (13 tests: skip no config, allowed, rejected, headers always set, 429 format, key extraction ip/xff/header/header-missing/claim/claim-no-ctx, fail open, composite key)

## Phase 6: Resilience (Circuit Breaker + Retry) — COMPLETE

- Task: Circuit breaker state machine + retry with exponential backoff + jitter, per-route wiring
- Key decisions:
  - No external deps; stdlib only
  - CB: three-state machine (Closed/Open/HalfOpen); fn() executes outside lock for concurrency
  - CB options: WithOnStateChange (for Prometheus gauge), WithClock (for testing)
  - Retry: body buffered into []byte, replayed via io.NopCloser(bytes.NewReader) per attempt
  - Backoff: initialDelay * multiplier^(attempt-1) * jitter where jitter ∈ [0.5, 1.5)
  - httptest.NewRecorder captures per-attempt response; only final success copied to real writer
  - CB opens on 500+ status; retryable codes configurable per route
  - Circuit open → 503, retries exhausted → 502, client disconnect → 499
  - WithCircuitBreaker(cb, proxy) for CB-only (no retry) wiring
  - Prometheus: gateway_circuit_breaker_state gauge updated via onStateChange callback in main.go
  - Registry/metrics moved before handler loop so CB callback can reference metrics
  - Context cancellation checked before and after sleep
- Files created:
  - internal/middleware/circuitbreaker/circuitbreaker.go (State machine, Execute, options)
  - internal/middleware/circuitbreaker/circuitbreaker_test.go (11 tests)
  - internal/middleware/retry/retry.go (Retry, WithCircuitBreaker, backoff, body buffering)
  - internal/middleware/retry/retry_test.go (15 tests + 2 integration)
- Files modified:
  - cmd/gateway/main.go (import CB+retry, per-route wiring, moved registry init before handler loop)
- All validations pass: gofmt, go test ./..., go vet ./...
